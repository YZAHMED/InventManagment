üé® Design Decisions I Made:

  • Consistent Header: Every page has the same title and navigation so users know where they are
  • Color-Coded Quantities: Green for high stock, yellow for medium, red for low - makes it easy to see what needs attention
  • Confirmation Dialogs: Added warning page for deletes because you can't undo that!
  • Empty State: When there are no items, I show a friendly message instead of just blank space
  • Clear Buttons: All actions are clearly labeled - "Add", "Edit", "Delete", etc.
  • Form Organization: Grouped related fields together and made required vs optional clear

üí≠ What I Learned About UI Design:

This was my first time creating wireframes! I learned that it's really important to think about the user's journey through the app. I tried to make sure every page has a clear purpose and that users can always get back to where they came from.

The hardest part was deciding how much information to show on each page. I didn't want to overwhelm users but I also wanted to make sure they had all the info they needed. I think the table layout for the main list works well because it's familiar to most people.

I also realized how important consistent styling is - using the same colors and button styles throughout makes the whole app feel more professional and easier to use.
